# feat-400.1 补强：官网 RAG 批处理面试题

## 1. 为什么 Embedding 批处理通常比逐条并发更合适？

OpenAI兼容Embedding协议原生接受字符串数组。批处理把 N 次HTTP握手、鉴权和响应解析压缩为 `ceil(N / batchSize)` 次，同时维持串行批次，不会像无界 `Promise.all` 一样突然打满provider限流、连接数和内存。

## 2. Provider批量响应为什么必须按 `index` 对齐？

协议不保证响应数组顺序与输入完全一致。按响应项 `index` 放回原chunk位置，才能避免向量与正文错配；缺失、重复或维度错误的项单独降级为NULL，不能污染同批有效向量。

## 3. 批量向量INSERT失败时如何保证正文不丢失？

一个多值INSERT语句在PostgreSQL中是原子的，任一向量非法会使整条语句失败，不会只写半批。捕获后用同一批正文、NULL embedding重试，保留稀疏检索能力；若NULL重试也失败则继续抛错，不能吞掉真实数据库故障。

## 4. 为什么批量写库时只借一次连接？

Embedding网络请求全部完成后再借PoolClient，随后多个INSERT批次复用该连接并立即归还。这样减少池获取/释放次数，也不会在等待外部API时长期占用连接池槽位。

## 5. 如何衡量批处理收益而不混入其他变量？

Embedding用固定provider往返延迟比较请求次数；数据库在同一连接、同一临时pgvector表上比较逐行和多值INSERT，并多轮取中位数。完整官网导入还包含抓取限速和图片下载，因此微基准加速比不能直接当作端到端加速比。
