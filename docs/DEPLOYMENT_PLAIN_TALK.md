# 部署方案大白话版

> 给非技术朋友 / 简历面试官 / 三个月后忘了细节的自己看。

---

我做了一个 RAG + Agent 的产品（智能营销文案生成器），最近把它部署上线了。整个部署方案听起来挺复杂，但用大白话讲其实就一句话：**前端放免费 CDN，后端放自己的小服务器，中间用一条"加密专线"把它们连起来。**

具体怎么搞？我分四块讲。

---

**第一块：前端放在 Vercel 上。**

Vercel 是国外一个免费的网站托管平台，你把 GitHub 仓库一连，它就自动帮你构建、部署、配 HTTPS 证书，全世界访问都很快。我项目里的前端是 Next.js，正好是 Vercel 主推的框架，部署起来一键搞定。免费的，不花钱。

**第二块：后端放在阿里云一台很小的服务器上。**

我买的是阿里云最便宜的 ECS，1 核 CPU、2GB 内存——大概一个月几十块钱。后端是 NestJS 应用、PostgreSQL 数据库、还有一个 Cloudflare 隧道客户端，三个东西用 Docker 打包，一条命令就能全部启动。

这里有个细节：原本我还想在这台机器上跑一个开源的 embedding 模型（叫 bge-m3，用来把文字转成向量，做语义检索的核心组件），但这玩意儿吃内存 600MB，我的小破服务器跑起来就 OOM 崩了。后来我换成了**调用智谱的云端 embedding API**，按调用量计费，演示用量一个月不到 1 块钱。**用钱换资源，比升级硬件划算。**

**第三块：中间用 Cloudflare 隧道连起来——这是整个方案最巧的地方。**

按常规做法，后端要让外网访问，必须开放 80 / 443 端口，配 HTTPS 证书，还要给域名做工信部备案（国内规定）。这一套下来又麻烦又有安全风险。

Cloudflare 提供了一个免费的服务叫 **Tunnel**，工作原理反过来：**不是别人来连你的服务器，而是你的服务器主动出去连 Cloudflare**——就像浏览器主动连百度一样，是出站连接，不需要开任何入站端口。

用户访问我的 API 网址 `api.retreevo.online`，DNS 把它指向 Cloudflare 的边缘节点（CF 在全球有几百个数据中心），Cloudflare 通过那条隧道把请求转发到我的小服务器上，回来再原路返回。**全程 HTTPS、自动证书续期、零运维**。

更妙的是：因为流量入口是 Cloudflare 的服务器（在境外），不是我的阿里云 ECS，所以**域名不需要备案**——这给个人开发者省了至少 2 周等审批的时间。

**第四块：前端怎么调后端？**

前端代码里写死一个 API 地址 `https://api.retreevo.online`。用户在浏览器里输个问题 → 前端 fetch 到这个地址 → CF 边缘 → 隧道 → 我的 ECS → API 处理（可能调智谱 LLM、查数据库、跑 RAG 检索）→ 结果原路返回。Agent 的实时思考过程用 SSE（Server-Sent Events）流回来，**用户能看到 Agent 一步步推理、调工具、查知识库的全过程**——这是项目的核心卖点：透明可观测。

---

**整体的好处？**

1. **便宜**：月成本不到 100 块（ECS 几十块 + 域名摊销 1 块 + API 调用费 1 块）
2. **稳**：Vercel 全球 CDN、Cloudflare 加持，前端访问飞快；后端用 Docker compose 一键起停，挂了 5 分钟回滚
3. **没运维**：HTTPS 自动、CDN 自动、备份用 cron 一行脚本搞定
4. **可演示**：HR 点链接 30 秒内能看到 UI，发消息能看到 Agent 实时跑

**整体的代价？**

1. 后端在国内 ECS，但流量绕 Cloudflare 海外节点，**延迟会增加 100-200ms**（对 demo 完全没影响，对真正的高频生产服务就要重新考虑）
2. 用了云端 embedding 后，**每次检索要花点钱**（虽然便宜到忽略不计）
3. 多个云平台拼接，**故障排查链路长**（一个请求要穿过 Vercel + CF + ECS 三层）

---

**总结一句话**：
**用 Vercel 解决前端 CDN 和 SSL 问题，用 Cloudflare Tunnel 解决后端公网暴露和备案问题，用云端 API 解决小内存服务器跑不起来重型模型的问题。每一项都是「现成的免费服务」，自己只管业务代码——这就是 2026 年个人开发者「以小博大」的最佳实践。**
